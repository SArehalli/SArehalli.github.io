Generalized Minesweeper

In Chapter 7, Problem 30 in *Siper's* Introduction to the Theory of Computation, Sipser instroduces a version of Minesweeper generalized to arbitrary graphs. The core of the game remains the same: The board consists of tiles which are either safe, or mines, with their nature initially hidden. The player then repeatedly selects tiles, revealing their true nature. If the tile is a mine, the player loses. If the tile is safe, the tile is labeled with the number of adjacent tiles that contain a mine.[1] The game is normally played on a grid, but Sipser's version is played on an arbitrary undirected graph with each tile being represented by a node. Of course, Sipser doesn't mention this game for the gameplay. He uses generalized minesweeper to pose a different kind of game --- a decision problem! In any real game of generalized minesweeper, we start by placing the mines and then reveal numbers of each vertex that are consistent with the underlying mine-to-vertex assignment. However, as a player, you could be suspicious that the numbers being shown to you are not consistent with the true mine-to-vertex assignment. This is, however, impossible to know as stated since you don't know the mine assignment. However, you can know if the board is inconsistent with any possible mine-to-vertex assignment.

This is exactly Sipser's Mine Consistency Problem: Given a partially solved board of generalized minesweeper, you must determine whether there is mine-to-vertex assignment that is consistent with the revealed numbers. Put another way, given a partially solved board, you have to determine whether the numbers revealed reflect a valid minesweeper board. Sipser asks you to show that this problem is NP-Complete --- both in class NP and an NP-Complete problem polynomial-time reduces to it. Showing it's in class NP is not-so tricky (consider what a good certificate should be for a polynomial-time verifier), but showing a reduction from 3-SAT is quite interesting!

Our high-level strategy here is to build gadgets, small pieces of our graph that correspond to units in a 3-SAT formula: variables/literals, clauses, and formulas. First, variables/literals!

Play around with the figure above to confirm your intuition: Selecting an unrevealed vertex toggles whether it contains a mine (indicated by the highlight), and the revealed vertex with be colored red when the mine assignment you've selected is inconsistent. You should see that the vertex with a 1 is only consistent when the assignment corresponds to a valid boolean assignment to the variable V! We have established that we have a valid assignment of mines to the minesweeper graph iff we have a valid assignment of boolean values to variables!

Now we have to handle clauses. Since we're using 3-SAT, we need a way to represent a 3-clause, or an OR of 3 literals. That is, we need there to be a valid mine-to-vertex assignment iff there is a valid assignment to our boolean variables that has our particular 3-clause evaluate to TRUE. Of course, this will require building a new gadget that utilizes out literal gadgets from before and has some revealed vertices to constrain our mine-consistency. Now, the fun part is how to turn this into a full representation of 3-SAT, which talks about the satsifiability of formulas that consist of a conjunction (i.e., an AND) of clauses. How do we do this? Simpler than you think!

The key here is realizing that if we reuse the same set of literal vertices, we end up needing to find a single assignment to those variables that satisfies every clause sub-graph! Thus, with this construction, we can show that if we can solve the minesweeper consistency problem, we can use that solution to solve 3-SAT: Simply take a 3-SAT formula (in CNF, as we assume), and use the construction above to generate a partially revealed board of generalized minesweeper. Then, we simply run our minesweeper consistency problem solution and it'll give us the correct answer to our original 3-SAT problem. To spell it out fully, there is a satisfying assignment to a 3-SAT formula F iff there is a satisfying mine-to-vertex assignment in the corresponding partial minesweeper board we construct! This tells us that 3-SAT reduces to the minesweeper consistency problem, but we must show that it polynomial-time reduces to the minesweeper consistency problem, so we have to demonstrate that this reduction happens in polynomial time. This last detail is boring though (constant work per variable, then constant work per clause to convert the formula to the graph, etc.).

Of course, I've stopped short of providing a full formal proof, which would have to show that the construction is correct for arbitrary 3-SAT formulae (If I were given a satisfiable formula, there exists a consistent mine-to-vertex assignment in the corresponding graph, and if I had a consistent mine-to-vertex assignment in the corresponding graph, I can construct a satisfying variable assignment for the original formula). That should certainly be doable (if a bit tedious) with a good intuition for the construction!

[1] Note that in standard minesweeper, if you click a tile with no adjacent mines, the game automatically flood-fills until all non-zero safe tiles are revealed. This is largely a convenience feature, as no new information is revealed through this process, so I don't replicate it here (also, Sipser doesn't mention it!).
